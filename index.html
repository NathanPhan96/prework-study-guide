Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An element refers to the entire element from opening tag to closing tag.
  • A tag only refers to what appears inside the angle brackets.
  • HTML is used to create the structure of a webpage.
  • HTML is one of the three cornerstone languages used in web development.
  • HTML elements work together to build the structure of a web page and provide a framework for the user to navigate the webpage.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A box-shadow CSS property adds shadow effects around an element's frame.
  • Any HTML element assigned to the class, card, will gain property assignments from the CSS.
  • CSS box model: every HTML element is essentially a box and each box has margins outside the element (element's borders, padding, and content).
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git branch: list your branches. a * will appear next to currently active branch
  • git log: show all commits in the current branch's history
  • git init: intialize an existing directory as a Git repository
  • git diff: diff of what is changed but not staged

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A function is a set of instructions that tells the computer how to perform a certian task.
  • A function is different from a loop/conditional statement because functions don't automatically execute when the JavaScript file is run.
  • Since control flow dictates how JavaScript files are executed from top to bottom, order is very important.
  • When you create a function, you are defining a block of code that performs a specific task and giving it a unique name.
